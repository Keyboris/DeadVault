First: what a deadman switch does:
- "If I don't check in within X time → trigger action"
- Transfer funds to heirs, reveal private keys/secrets, execute predefined transactions
- Multisig is about WHO approves actions
- Deadman switch is about WHEN actions trigger
- Combining them = conditional shared control

Fail-safe inheritance system:
- Safe has 3 owners: You + 2 trusted people, Threshold = 2
- If you're inactive → auto-transfer proposal is created
- Others can recover access only if you're gone

2. DeadmanModule.sol — Deadman switch module:
   - Owners must checkIn() before deadline (lastCheckIn + inactivityPeriod)
   - ALL owners miss check-in → anyone calls startGracePeriod()
   - Grace expires without check-in → anyone calls trigger() → funds to beneficiaries
   - Any single owner checking in cancels grace period
   - Uses execTransactionFromModule to send funds (no signatures needed)

3. MultiSigFactory.sol — Deploys wallet + module pair in one transaction
